## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Backend API → ProcessNode data | `node.brief_description` and `node.name` rendered into the DOM via React — content comes from the Excel parse |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | Hover popover rendering `node.brief_description` | accept | Content is read-only data from the eTOM Excel file (no user input, no PII). React automatically escapes string content rendered via JSX — XSS not possible through this path. |
| T-01-02 | Tampering | `title={vg}` attribute on VG chip spans | accept | `vg` value is drawn from `node.vertical_groups` which is parsed from the Excel at startup and never written by users. React escapes attribute values. |
| T-01-03 | Denial of Service | Unbounded `setTimeout` timer if component unmounts during 250ms delay | mitigate | Use `useRef` to store the timer and call `clearTimeout` in the `onMouseLeave` handler. Component unmount edge case: if needed, add `useEffect` cleanup — but since `hoverTimerRef` only calls `setHovered` (no network/side-effects), React will silently ignore setState on unmounted components (no crash, no leak). Accept residual risk. |
</threat_model>
